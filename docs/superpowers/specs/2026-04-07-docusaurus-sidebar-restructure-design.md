# Desain Restrukturisasi Sidebar dan Docs `docusaurus-site`

Tanggal: 2026-04-07
Proyek: `docusaurus-site`

## Latar Belakang

`docusaurus-site` saat ini masih memakai sidebar autogenerated, sementara `panduan-microservice-nestjs` sudah memakai sidebar eksplisit yang lebih enak dipakai untuk course berurutan.

Pengguna menyukai pola sidebar di `panduan-microservice-nestjs` dan ingin `docusaurus-site` mengikuti pola yang sama, termasuk:

- struktur sidebar
- struktur folder dokumen
- slug/path URL dokumen

Namun ada satu pengecualian penting: `docs/index.md` tetap dipertahankan dan tidak dipindah ke pola `getting-started`.

## Tujuan

- Mengubah sidebar `docusaurus-site` dari autogenerated menjadi eksplisit.
- Menyusun ulang folder docs agar mengikuti pola `Getting Started`, `Learning Path`, dan `Reference`.
- Mengubah slug/path URL dokumen agar konsisten dengan pola `panduan-microservice-nestjs`.
- Mempertahankan `docs/index.md` sebagai halaman root docs.

## Non-Tujuan

- Tidak menulis ulang seluruh materi course Go.
- Tidak mengubah isi homepage situs pada task ini.
- Tidak mengubah `docs/index.md` menjadi `getting-started/pengenalan.md`.
- Tidak membuat redirect compatibility untuk slug lama kecuali benar-benar dibutuhkan oleh build.

## Pendekatan yang Dipilih

Pendekatan yang dipilih adalah `structure-first`.

Artinya:

- file dipindah ke struktur folder baru
- front matter/slugs diperbarui mengikuti struktur baru
- sidebar dibuat eksplisit berdasarkan struktur baru

Pendekatan ini dipilih karena paling konsisten dengan target akhir dan menghindari setengah migrasi.

## Risiko dan Mitigasi

Risiko:

- link internal bisa putus setelah file dipindah
- front matter `pagination_prev` dan `pagination_next` bisa mengarah ke path lama
- sidebar bisa sinkron tetapi slug lama tidak lagi valid

Mitigasi:

- update internal links sekaligus saat pemindahan file
- review front matter seluruh file yang dipindah
- jalankan build penuh Docusaurus setelah migrasi

## Rencana Verifikasi

Sebelum pekerjaan dinyatakan selesai:

- `npm run -s build` harus sukses
- sidebar harus menampilkan struktur eksplisit baru
- semua dokumen utama harus bisa di-resolve tanpa broken link
- `docs/index.md` tetap menjadi halaman docs root

## Kriteria Selesai

Pekerjaan dianggap selesai jika:

- `docusaurus-site` memakai pola sidebar seperti `panduan-microservice-nestjs`
- struktur folder docs mengikuti pola baru
- slug/path URL dokumen mengikuti pola baru
- `docs/index.md` tetap dipertahankan di root
